💡 Decisiones Arquitecturales
Stateless
Sin repositorio ni acceso a DB. Todo el estado vive en el módulo de Invoicing (sync_logs). Softpymes solo procesa y responde.
URL dinámica
No hay URL hardcoded. Se resuelve desde integration_types (baseURL vs baseURLTest) por cada request.
Token cache por URL
Prod y testing pueden usar endpoints distintos con tokens diferentes simultáneamente en el mismo proceso.
Cash receipt no-fatal
Si el recibo de caja falla, la factura ya está creada. El pago se puede registrar manualmente después.
Idempotencia en retry
Solo en retry: busca factura existente por order_id en comment antes de crear. Evita duplicados.
Auditoría completa
Cada llamada HTTP captura request/response completo. Se almacena en sync_logs para debugging y compliance.
